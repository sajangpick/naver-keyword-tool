# 서버 시작 배치 파일 문제 해결 가이드

## 🔍 문제 진단

### 배치 파일을 더블클릭했는데 아무 반응이 없는 경우

#### 1단계: 배치 파일이 실행되는지 확인

**방법 1: 테스트 배치 파일 사용**
- `서버시작_테스트.bat` 파일을 더블클릭하세요
- 이 파일은 각 단계마다 일시정지하므로 어디서 문제가 발생하는지 확인할 수 있습니다

**방법 2: 명령 프롬프트에서 직접 실행**
1. `서버시작.bat` 파일이 있는 폴더로 이동
2. 파일 탐색기에서 주소창에 `cmd` 입력 후 Enter
3. 명령 프롬프트 창에서 `서버시작.bat` 입력 후 Enter
4. 오류 메시지가 표시되면 그 내용을 확인하세요

#### 2단계: Windows 보안 설정 확인

배치 파일 실행이 차단되어 있을 수 있습니다:

1. **파일 속성 확인**
   - `서버시작.bat` 파일을 우클릭 → 속성
   - "차단 해제" 버튼이 있으면 클릭

2. **스마트 스크린 확인**
   - Windows 보안 설정에서 스마트 스크린 확인
   - 배치 파일 실행 허용 설정

#### 3단계: Node.js 설치 확인

명령 프롬프트에서 다음 명령어 실행:
```bash
node --version
```

- **오류가 나면**: Node.js가 설치되지 않았거나 PATH에 등록되지 않았습니다
- **해결 방법**: Node.js를 설치하거나 PATH 환경변수에 추가하세요

#### 4단계: 파일 위치 확인

배치 파일이 올바른 폴더에 있는지 확인:
- `server.js` 파일과 같은 폴더에 있어야 합니다
- 폴더 경로에 한글이나 특수문자가 많으면 문제가 될 수 있습니다

## 🛠️ 해결 방법

### 방법 1: 테스트 배치 파일 사용 (추천)

1. `서버시작_테스트.bat` 파일을 더블클릭
2. 각 단계에서 일시정지되므로 문제 지점 확인 가능
3. 오류 메시지를 읽고 해결

### 방법 2: 명령 프롬프트에서 직접 실행

1. 파일 탐색기에서 `서버시작.bat` 파일이 있는 폴더 열기
2. 주소창에 `cmd` 입력 후 Enter
3. 명령 프롬프트 창이 열리면:
   ```bash
   서버시작.bat
   ```
4. 오류 메시지 확인

### 방법 3: 수동으로 서버 시작

명령 프롬프트에서:
```bash
cd "C:\Users\오다쉐프\Desktop\naver-keyword-tool-main\naver-keyword-tool-main"
node server.js
```

## ❓ 자주 발생하는 문제

### Q: "Node.js를 찾을 수 없습니다" 오류
**해결**: Node.js 설치 확인
- [Node.js 공식 사이트](https://nodejs.org/)에서 다운로드
- 설치 후 컴퓨터 재시작

### Q: "server.js 파일을 찾을 수 없습니다" 오류
**해결**: 올바른 폴더에 있는지 확인
- 배치 파일과 `server.js`가 같은 폴더에 있어야 합니다

### Q: 배치 파일을 실행하면 바로 창이 닫힙니다
**해결**: 명령 프롬프트에서 실행
- 파일 탐색기 주소창에 `cmd` 입력
- 명령 프롬프트에서 배치 파일 실행

### Q: "권한이 없습니다" 오류
**해결**: 관리자 권한으로 실행
- 배치 파일을 우클릭 → "관리자 권한으로 실행"

## 📞 추가 도움이 필요한 경우

문제가 계속되면 다음 정보를 확인하세요:
1. Windows 버전
2. Node.js 버전 (`node --version`)
3. 오류 메시지 전체 내용
4. 배치 파일이 있는 폴더 경로
