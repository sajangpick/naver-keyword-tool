# 로컬 테스트 간단 가이드

## 🚨 문제: "사이트에 연결할 수 없음" 오류

`localhost:5500`으로 접속했는데 연결이 안 되는 경우, **Live Server가 실행되지 않았거나 포트가 다릅니다.**

---

## ✅ 해결 방법 1: VS Code Live Server 실행 (가장 간단)

### 1단계: VS Code에서 파일 열기
1. VS Code를 엽니다
2. `payment.html` 파일을 엽니다

### 2단계: Live Server 실행
1. `payment.html` 파일에서 **마우스 오른쪽 버튼 클릭**
2. 메뉴에서 **"Open with Live Server"** 선택
3. 자동으로 브라우저가 열립니다 (보통 `http://127.0.0.1:5501/...` 또는 `http://localhost:5501/...`)

### 3단계: 데모 모드 URL 확인
브라우저 주소창에 다음을 입력:
```
http://127.0.0.1:5501/payment.html?demo=true
```
또는
```
http://localhost:5501/payment.html?demo=true
```

**참고:** Live Server는 보통 **5501 포트**를 사용합니다 (5500이 아님)

---

## ✅ 해결 방법 2: Node.js 서버 사용 (API 테스트용)

### 1단계: 터미널 열기
1. VS Code에서 `Ctrl + `` (백틱) 키를 누르거나
2. 상단 메뉴: **터미널 → 새 터미널**

### 2단계: 서버 실행
터미널에 다음 명령어 입력:
```bash
node server.js
```

### 3단계: 브라우저에서 접속
브라우저 주소창에 입력:
```
http://localhost:3003/payment.html?demo=true
```

**참고:** 
- 서버가 실행되면 터미널에 "서버가 시작되었습니다" 메시지가 보입니다
- 터미널 창을 닫으면 서버가 종료됩니다

---

## 🔍 포트 확인 방법

### Live Server 포트 확인
1. VS Code 하단 우측에 포트 번호가 표시됩니다
2. 보통 `5501` 또는 `5500`입니다
3. 표시된 포트 번호로 접속하세요

### Node.js 서버 포트 확인
1. 터미널에서 서버 실행 시 포트 번호가 표시됩니다
2. 보통 `3003`입니다
3. 예: `🚀 서버가 시작되었습니다! 포트: 3003`

## ⚠️ 자주 발생하는 오류

### 오류 1: "포트가 이미 사용 중입니다"
**해결:** 
- 다른 터미널에서 실행 중인 서버가 있는지 확인
- `Ctrl + C`로 기존 서버 종료 후 다시 시작

### 오류 2: "node를 찾을 수 없습니다"
**해결:**
- Node.js가 설치되지 않았습니다
- [Node.js 공식 사이트](https://nodejs.org/)에서 다운로드 및 설치

### 오류 3: Live Server가 실행되지 않음
**해결:**
- VS Code 확장 프로그램에서 "Live Server" 설치 확인
- VS Code 재시작 후 다시 시도

**데모 모드만 확인하려면:** Live Server 사용 (방법 1)  
**API 기능도 테스트하려면:** Node.js 서버 사용 (방법 2)

## 🆘 여전히 안 될 때

1. **브라우저 주소 확인**
   - `http://127.0.0.1:5501/...` (Live Server)
   - `http://localhost:3003/...` (Node.js 서버)

2. **포트 번호 확인**
   - VS Code 하단 우측 포트 번호 확인
   - 터미널 메시지에서 포트 번호 확인

3. **파일 경로 확인**
   - `payment.html` 파일이 프로젝트 루트에 있는지 확인

4. **오류 메시지 스크린샷**
   - 오류 메시지를 캡처하여 관리자에게 문의
