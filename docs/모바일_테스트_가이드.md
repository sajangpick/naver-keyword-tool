# 모바일 테스트 가이드

배포하지 않고 실제 모바일 기기에서 테스트하는 방법입니다.

## 방법 1: 같은 네트워크에서 로컬 서버 접속 (가장 간단)

### 준비사항
- PC와 모바일 기기가 같은 WiFi 네트워크에 연결되어 있어야 합니다.

### 단계

1. **PC에서 로컬 서버 실행**
```bash
# 프로젝트 폴더에서
pnpm start
# 또는
node server.js
```

서버가 실행되면 다음과 같은 메시지가 보입니다:
```
Server is running on port 3003
```

2. **PC의 IP 주소 확인**

**Windows:**
```powershell
# PowerShell에서 실행
ipconfig
```
`IPv4 주소`를 찾으세요. 예: `192.168.0.100`

**Mac/Linux:**
```bash
ifconfig | grep "inet "
```
또는
```bash
ip addr show
```

3. **모바일 기기에서 접속**

모바일 브라우저(Chrome, Safari 등)에서 다음 주소로 접속:
```
http://192.168.0.100:3003
```
(PC의 실제 IP 주소로 변경하세요)

### 장점
- 추가 설치 없이 바로 사용 가능
- 빠르고 간단함

### 단점
- PC와 모바일이 같은 WiFi에 있어야 함
- 방화벽 설정에 따라 접속이 안 될 수 있음

---

## 방법 2: ngrok 사용 (인터넷 어디서든 접속 가능)

### 준비사항
- ngrok 계정 (무료)
- ngrok 설치

### 단계

1. **ngrok 설치**

**Windows:**
- [ngrok 공식 사이트](https://ngrok.com/download)에서 다운로드
- 또는 Chocolatey 사용: `choco install ngrok`

**Mac:**
```bash
brew install ngrok
```

2. **ngrok 계정 가입 및 토큰 설정**
- https://dashboard.ngrok.com 에서 무료 계정 생성
- 토큰 복사 후 터미널에서 실행:
```bash
ngrok config add-authtoken YOUR_TOKEN_HERE
```

3. **로컬 서버 실행**
```bash
pnpm start
```

4. **새 터미널에서 ngrok 실행**
```bash
ngrok http 3003
```

5. **ngrok이 제공하는 URL 사용**
터미널에 다음과 같은 정보가 나타납니다:
```
Forwarding  https://abc123.ngrok-free.app -> http://localhost:3003
```

이 URL을 모바일 브라우저에서 접속하면 됩니다!

### 장점
- 인터넷 어디서든 접속 가능
- HTTPS 지원
- 실제 배포 환경과 유사

### 단점
- 무료 버전은 세션 시간 제한 있음 (2시간)
- URL이 매번 변경됨

## 방법 3: Chrome Remote Debugging (가장 정확한 디버깅)

### 준비사항
- Android 기기와 Chrome 브라우저
- USB 케이블

### 단계

1. **Android 기기에서 개발자 옵션 활성화**
   - 설정 → 디바이스 정보 → 빌드 번호 7번 탭
   - 설정 → 개발자 옵션 → USB 디버깅 활성화

2. **USB로 PC에 연결**

3. **PC에서 Chrome 열기**
   - 주소창에 `chrome://inspect` 입력
   - "Remote devices" 섹션에서 기기 확인

4. **로컬 서버 실행**
```bash
pnpm start
```

5. **PC에서 포트 포워딩 설정**
   - Chrome의 `chrome://inspect` 페이지에서
   - "Port forwarding" 설정
   - Port: `3003`, IP: `localhost:3003` 추가

6. **Android Chrome에서 접속**
   - `localhost:3003` 또는 `127.0.0.1:3003` 입력

### 장점
- 실제 디바이스에서 정확한 테스트
- Chrome DevTools로 디버깅 가능
- 실시간 요소 검사 및 수정 가능

## 방법 4: 간단한 HTTP 서버 (정적 파일만 테스트)

HTML 파일만 빠르게 확인하고 싶을 때:

### Python 사용 (가장 간단)
```bash
# Python 3가 설치되어 있다면
python -m http.server 8000
```

### Node.js http-server 사용
```bash
# 설치
pnpm add -g http-server

# 실행 (프로젝트 폴더에서)
http-server -p 8000
```

그 다음 방법 1과 같이 PC의 IP 주소로 접속:
```
http://192.168.0.100:8000
```

### 주의사항
- API 호출이 작동하지 않을 수 있음 (서버가 필요함)
- 정적 HTML/CSS/JS만 테스트 가능

---

## 추천 방법

**빠른 확인:** 방법 1 (같은 네트워크)
**정확한 테스트:** 방법 3 (Chrome Remote Debugging)
**외부 접속 필요:** 방법 2 (ngrok)

---

## 문제 해결

### "연결할 수 없음" 에러
- PC 방화벽 확인: Windows 방화벽에서 포트 3003 허용
- 같은 WiFi 네트워크인지 확인
- IP 주소가 올바른지 확인

### 서버가 안 뜸
- 포트가 이미 사용 중인지 확인: `netstat -ano | findstr :3003` (Windows)
- 다른 포트 사용: `PORT=3004 pnpm start`

### ngrok이 작동 안 함
- 토큰이 제대로 설정되었는지 확인
- 방화벽이나 회사 네트워크에서 막힐 수 있음
