## 🚀 1단계: Vercel 배포 URL 확인하기

먼저 **실제 배포된 주소**를 확인해야 합니다.

### 방법 1: Vercel 대시보드에서 확인
```
1. Vercel 대시보드 접속 (https://vercel.com/dashboard)
2. 프로젝트 클릭
3. "Domains" 탭에서 URL 확인
   예: https://sajangpick.vercel.app
```

### 방법 2: Git에서 확인
```bash
# 터미널에서 실행
git remote -v
# vercel 관련 URL이 보입니다
```

### ⚠️ 중요: URL 업데이트 필요
`sitemap.xml`과 `robots.txt` 파일에 있는 **`https://your-domain.vercel.app`** 를 실제 URL로 바꿔야 합니다!

**예시:**
```xml
<!-- 변경 전 -->
<loc>https://your-domain.vercel.app/</loc>

<!-- 변경 후 (실제 URL로) -->
<loc>https://sajangpick.vercel.app/</loc>
```

## 🔧 2단계: sitemap.xml과 robots.txt 수정하기

### 2-1. 실제 URL 확인 후 수정

1. **Vercel 대시보드**에서 실제 배포 URL 복사
   - 예: `https://sajangpick.vercel.app`

2. **sitemap.xml** 파일 열기
   - `Ctrl+F`로 `your-domain` 검색
   - 모두 실제 도메인으로 변경

3. **robots.txt** 파일 열기
   - 마지막 줄 `Sitemap:` 부분을 실제 URL로 변경

### 2-2. GitHub에 배포
```bash
# 파일 저장 후
git add sitemap.xml robots.txt
git commit -m "feat: 네이버 서치어드바이저용 sitemap과 robots.txt 추가"
git push origin main
```

Vercel이 자동으로 배포합니다! (약 1~2분 소요)

### 2-3. 배포 확인
```
브라우저에서 접속:
https://[실제도메인]/sitemap.xml
https://[실제도메인]/robots.txt

정상적으로 보이면 성공! ✅
```

## 🔍 트러블슈팅

### ❌ 소유 확인 실패 시

**문제 1**: "파일을 찾을 수 없습니다"
```
해결:
1. Vercel 배포 완료 확인 (Vercel 대시보드)
2. 브라우저에서 직접 접속해보기
3. 캐시 삭제 후 재시도 (Ctrl+F5)
4. 파일명이 정확한지 확인 (대소문자 구분!)
```

**문제 2**: "접근 권한 없음"
```
해결:
1. robots.txt에서 HTML 파일 차단 확인
2. Vercel 배포 설정 확인
3. CORS 설정 확인
```

**문제 3**: "사이트맵 오류"
```
해결:
1. sitemap.xml 문법 확인 (XML 검증기 사용)
2. URL이 모두 실제 도메인인지 확인
3. lastmod 날짜 형식 확인 (YYYY-MM-DD)
```
