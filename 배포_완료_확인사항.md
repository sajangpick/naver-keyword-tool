# ✅ 배포 완료 후 확인 사항

## 1. Render 서버 상태 확인

### 헬스체크 테스트
브라우저에서 직접 접속:
```
https://naver-keyword-tool.onrender.com/health
```

**예상 결과:**
- ✅ `{"status":"ok"}` → 서버 정상 작동
- ❌ `502 Bad Gateway` → 서버 다운 또는 에러
- ❌ `503 Service Unavailable` → 데이터베이스 연결 실패

## 2. 영상 생성 테스트

### 테스트 방법
1. https://sajangpick.co.kr/shorts-editor 접속
2. 이미지 업로드
3. 메뉴 정보 입력
4. 스타일 선택
5. "영상 생성" 버튼 클릭

### 예상 동작
1. **1순위**: Gemini Veo 3.1 API 시도 (제미나이 키 사용)
2. **실패 시**: Runway API 폴백
3. **실패 시**: Runway HTTP API 폴백

## 3. Render 로그 확인

Render 대시보드 → "Logs" 탭에서 확인:

### 정상 작동 시 로그
```
✅ Supabase 클라이언트 초기화 성공
✅ Gemini API 키 확인 완료
✅ Runway SDK 초기화 성공
🚀 서버가 포트 10000에서 실행 중입니다
```

### 에러 발생 시 로그
- `SyntaxError` → 문법 오류 (이미 수정됨)
- `GEMINI_API_KEY is not set` → 환경변수 문제
- `Cannot connect to database` → Supabase 연결 문제
- `BadRequestError: 400` → Runway API 모델 이름 문제 (이미 수정됨)

## 4. 문제 해결 체크리스트

### 문제 1: 여전히 502 에러
**확인:**
- [ ] Render 서버 상태가 "Live"인지 확인
- [ ] 헬스체크가 정상인지 확인
- [ ] 로그에 에러가 없는지 확인

**해결:**
- Render 서버 재시작
- 환경변수 재확인

### 문제 2: "Gemini Veo API가 아직 공개되지 않았습니다"
**원인**: Google Veo API 엔드포인트가 다르거나 아직 공개되지 않음

**해결:**
- 자동으로 Runway API로 폴백됨
- 정상 작동함 (제미나이 Veo는 나중에 사용 가능할 때까지 대기)

### 문제 3: "영상 생성 실패"
**확인:**
- [ ] Render 로그에서 실제 에러 메시지 확인
- [ ] Runway API 키가 설정되어 있는지 확인
- [ ] Supabase 테이블(shorts_videos)이 생성되어 있는지 확인
