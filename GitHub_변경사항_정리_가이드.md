# GitHub Desktop에서 변경사항 정리하기

## 🎯 목표
- 어제 수정한 파일들의 변경사항 되돌리기
- 오늘 방금 수정한 파일만 커밋하기

---

## 📋 단계별 가이드

### 1단계: 어제 수정한 파일들의 변경사항 되돌리기

**방법 A: 개별 파일 되돌리기 (추천)**

1. GitHub Desktop에서 변경사항 목록 확인
2. **어제 수정한 파일**을 하나씩 선택
3. 각 파일을 **우클릭** → **"Discard changes"** 선택
   - 또는 파일을 선택한 후 오른쪽 상단의 **"Discard"** 버튼 클릭
4. 확인 메시지가 나오면 **"Discard changes"** 클릭
5. 모든 어제 파일에 대해 반복

**방법 B: 모든 변경사항 한 번에 되돌리기**

⚠️ **주의**: 이 방법은 모든 변경사항을 되돌립니다!

1. GitHub Desktop 메인 화면에서
2. **Repository** → **Discard All Changes** 선택
3. 확인 메시지에서 **"Discard All Changes"** 클릭

---

### 2단계: 오늘 수정한 파일 확인하기

1. 변경사항을 되돌린 후
2. GitHub Desktop에서 새로고침 (F5 또는 Repository → Refresh)
3. 남아있는 변경사항이 **오늘 수정한 파일**인지 확인

---

### 3단계: 오늘 수정한 파일만 커밋하기

1. 변경사항 목록에서 **오늘 수정한 파일만** 체크되어 있는지 확인
2. 커밋 메시지 입력
   - 예: "오늘 수정한 내용"
3. **"Commit to main"** 클릭
4. **"Push origin"** 클릭

---

## 🔍 파일 수정 시간 확인하는 방법

### Windows 탐색기에서 확인:

1. GitHub Desktop에서 **Repository** → **Show in Explorer** 클릭
2. 파일을 **우클릭** → **속성** 클릭
3. **"수정한 날짜"** 확인

### 또는 GitHub Desktop에서:

1. 파일을 선택하면 오른쪽에 diff가 표시됩니다
2. 파일 목록에서 파일을 우클릭 → **"Show in Explorer"** 클릭
3. 탐색기에서 파일 속성 확인

---

## ⚠️ 주의사항

- **Discard changes**는 변경사항을 **완전히 삭제**합니다
- 되돌린 변경사항은 복구할 수 없습니다
- 중요한 변경사항이 있다면 먼저 백업하세요

---

## 💡 팁

**변경사항을 되돌리기 전에:**
- 파일을 다른 곳에 복사해두면 안전합니다
- 또는 브랜치를 만들어서 보관할 수도 있습니다

**브랜치로 보관하는 방법:**
1. **Branch** → **New Branch** 클릭
2. 브랜치 이름: `backup-yesterday-changes`
3. 커밋 메시지 입력 후 커밋
4. main 브랜치로 돌아가서 변경사항 되돌리기
