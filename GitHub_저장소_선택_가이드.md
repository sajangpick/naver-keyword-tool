# GitHub Desktop 저장소 선택 가이드

## 🤔 왜 같은 이름의 저장소가 여러 개 보이나요?

같은 저장소를 **다른 폴더에 여러 번 클론**했을 가능성이 높습니다.

### 가능한 이유:
1. **다른 경로에 저장소를 클론함**
   - 예: `C:\Users\오다쉐프\Documents\GitHub\naver-keyword-tool`
   - 예: `C:\Users\오다쉐프\Desktop\naver-keyword-tool`
   - 예: `C:\Users\오다쉐프\Desktop\naver-keyword-tool-main\naver-keyword-tool-main`

2. **각 저장소의 상태가 다를 수 있음**
   - 최신 상태인 저장소
   - 오래된 상태인 저장소
   - 경로 문제가 있는 저장소 (경고 아이콘)

---

## ⚠️ 경고 아이콘이 있는 저장소

**경고 아이콘(⚠️)이 있는 저장소는:**
- 저장소 경로가 더 이상 존재하지 않거나
- 폴더가 삭제되었거나 이동되었을 수 있습니다
- 이런 저장소는 사용하지 마세요

---

## ✅ 올바른 저장소 선택 방법

### 1단계: 각 저장소의 경로 확인하기

1. 각 `naver-keyword-tool` 저장소를 하나씩 클릭해보세요
2. **Repository** → **Show in Explorer** 클릭
3. 폴더가 정상적으로 열리는지 확인
4. 경로를 메모해두세요

### 2단계: 최신 상태인 저장소 찾기

**확인 방법:**
1. 저장소를 선택하면 오른쪽에 상태가 표시됩니다
2. "Last fetched" 시간 확인
3. "Pull origin" 또는 "Push origin" 버튼이 있는지 확인
4. 변경사항이 표시되는지 확인

### 3단계: 올바른 저장소 선택

**선택 기준:**
- ✅ 경고 아이콘이 없는 저장소
- ✅ 폴더가 정상적으로 열리는 저장소
- ✅ 최근에 업데이트된 저장소
- ✅ 변경사항이 정상적으로 표시되는 저장소

---

## 🎯 추천 방법

### 방법 1: 경고 없는 저장소 선택

1. 경고 아이콘(⚠️)이 **없는** 저장소를 선택하세요
2. 선택 후 **Repository** → **Show in Explorer** 클릭
3. 폴더가 정상적으로 열리면 그 저장소를 사용하세요

### 방법 2: 모든 저장소 정리하기

**불필요한 저장소 제거:**
1. 경고 아이콘이 있는 저장소를 우클릭
2. **"Remove"** 선택
3. GitHub Desktop에서만 제거됩니다 (실제 폴더는 남아있음)

**하나만 남기기:**
- 가장 최신이고 정상적인 저장소 하나만 남기세요
- 나머지는 GitHub Desktop에서 제거해도 됩니다

---

## 💡 팁

**하나의 저장소만 사용하는 것이 좋습니다:**
- 여러 저장소를 사용하면 혼란스럽습니다
- 하나의 저장소만 유지하고 나머지는 제거하세요
- 실제 파일은 삭제되지 않고, GitHub Desktop에서만 제거됩니다
