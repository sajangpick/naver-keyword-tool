# 서버 실행 및 테스트 가이드

## 문제 원인
404 오류는 **서버가 실행되지 않아서** 발생합니다.
프론트엔드는 `http://localhost:3003`로 API를 호출하지만, 서버가 실행되지 않으면 404 오류가 발생합니다.

## 해결 방법

### 1단계: 서버 실행 확인
서버가 실행 중인지 확인하세요:
- 터미널 창을 열고 `node server.js` 명령어를 실행했는지 확인
- 또는 `서버시작.bat` 파일을 더블클릭했는지 확인

### 2단계: 서버 실행하기

**방법 1: 배치 파일 사용 (가장 쉬움)**
1. 프로젝트 폴더에서 `서버시작.bat` 파일을 더블클릭
2. 터미널 창이 열리고 서버가 시작됩니다
3. "🚀 통합 API 서버가 시작되었습니다!" 메시지가 보이면 성공

**방법 2: 터미널에서 직접 실행**
1. 프로젝트 폴더로 이동
2. `node server.js` 명령어 실행
3. 서버가 포트 3003에서 시작됩니다

### 3단계: 서버 실행 확인
브라우저에서 다음 URL을 열어보세요:
- `http://localhost:3003/health`
- "status": "OK" 메시지가 보이면 서버가 정상 실행 중입니다

### 4단계: 연관 키워드 API 테스트
브라우저에서 다음 URL을 열어보세요:
- `http://localhost:3003/api/related-keywords?seed=테스트`
- JSON 응답이 보이면 API가 정상 작동합니다

## 주의사항
- 서버를 실행한 터미널 창을 닫지 마세요 (서버가 종료됩니다)
- 서버를 종료하려면 터미널에서 `Ctrl+C`를 누르세요
- 서버가 실행 중이어야 프론트엔드에서 API를 호출할 수 있습니다

## 문제 해결 체크리스트
- [ ] 서버가 실행 중인가요? (`http://localhost:3003/health` 확인)
- [ ] 포트 3003이 다른 프로그램에 사용되고 있지 않은가요?
- [ ] `.env` 파일에 네이버 API 키가 설정되어 있나요?
- [ ] `node_modules` 폴더가 있고 패키지가 설치되어 있나요? (`pnpm install` 실행)
