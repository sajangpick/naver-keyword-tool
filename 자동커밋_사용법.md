# 자동 커밋 스크립트 사용법

## 📋 개요

파일을 수정한 후 자동으로 Git 커밋과 푸시를 해주는 스크립트입니다.

## 🚀 사용 방법

### 방법 1: 배치 파일 (더블클릭)

1. **`auto-commit.bat`** 파일을 더블클릭합니다
2. 변경사항이 있으면 자동으로 커밋하고 푸시합니다
3. 변경사항이 없으면 "변경사항이 없습니다" 메시지가 표시됩니다

### 방법 2: PowerShell 스크립트

1. PowerShell을 엽니다
2. 프로젝트 폴더로 이동합니다
3. 다음 명령어를 실행합니다:
   ```powershell
   .\auto-commit.ps1
   ```

## ✨ 기능

- ✅ 변경사항 자동 감지
- ✅ 모든 파일 자동 추가 (`git add .`)
- ✅ 자동 커밋 메시지 생성 (날짜/시간 포함)
- ✅ 자동 푸시 (`git push origin main`)
- ✅ Vercel 자동 배포 트리거

## 📝 커밋 메시지 형식

자동으로 생성되는 커밋 메시지:
```
Auto commit: 2024-11-18 14:30:25
```

## ⚠️ 주의사항

1. **변경사항이 있을 때만 커밋됩니다**
   - 파일을 수정하지 않으면 커밋하지 않습니다

2. **푸시 실패 시**
   - GitHub Desktop에서 수동으로 푸시해주세요
   - 네트워크 문제나 인증 문제일 수 있습니다

3. **커밋 메시지 커스터마이징**
   - `auto-commit.bat` 파일을 열어서 `COMMIT_MSG` 부분을 수정하세요

## 🔧 고급 사용법

### 커스텀 커밋 메시지 사용

`auto-commit.bat` 파일을 열고 다음 부분을 수정하세요:

```batch
REM 자동 커밋 메시지 생성
set COMMIT_MSG=원하는 커밋 메시지
```

또는 PowerShell 스크립트에서:

```powershell
$commitMsg = "원하는 커밋 메시지"
```

## 📌 팁

- 파일을 수정한 후 바로 `auto-commit.bat`를 실행하세요
- 여러 파일을 수정한 후 한 번에 커밋할 수 있습니다
- 스크립트를 실행하면 자동으로 Vercel 배포가 시작됩니다
